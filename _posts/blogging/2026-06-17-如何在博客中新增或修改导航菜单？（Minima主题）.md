> 这是“博客搭建系列”的第五篇文章。前面我们解决了导航不显示的问题，最终切换到 Minima 主题并使用 `header_pages` 配置实现了导航。这一篇专门讲：**以后我想在导航栏加一个新页面（比如“标签”、“友链”、“归档”等），应该怎么操作？** 同时也会教你**如何修改导航栏里每个项目的显示名称**。这是一份面向未来的完整操作指南。

---

## 适用场景

你已经有一个基于 Minima 主题的 GitHub Pages 博客，并且已经配置好了基本的导航（如“首页”、“关于我”、“分类”）。现在你想：

- 在顶部导航栏增加一个新页面（如“标签”、“友链”、“归档”、“项目”等）
- 或者修改导航栏里某个项目的显示名称（比如把“分类”改成“文章归类”）

这篇文章会一次性教会你这两件事。

---

## 核心原理

在 Minima 主题中，导航菜单是通过 `_config.yml` 中的 `header_pages` 配置项自动生成的。你只需要：

1. **创建一个新的 `.md` 文件**（比如 `tags.md`），放在仓库根目录。
2. **在该文件的头部（Front Matter）指定 `layout: page` 和 `permalink`**。
3. **在 `_config.yml` 的 `header_pages` 列表中添加这个文件名**。
4. **提交推送**，导航栏就会自动更新。

整个过程**不需要修改任何 HTML 或 CSS 代码**。

---

## 详细操作步骤（以新增“标签”页为例）

### 第一步：创建页面文件

在仓库根目录下，新建一个文件，命名为 `tags.md`（文件名可以自定义，但建议用英文小写+连字符）。

### 第二步：编写页面内容

在 `tags.md` 中写入以下内容（这是一个完整的标签归档页面模板）：

```markdown
---
layout: page
title: 标签
permalink: /tags/
---

## 所有标签

{% assign tags = site.tags | sort %}
{% for tag in tags %}
  <h3 id="{{ tag[0] | slugify }}">{{ tag[0] }}</h3>
  <ul>
    {% for post in tag[1] %}
      <li>
        <a href="{{ post.url | relative_url }}">{{ post.title }}</a>
        <span style="color:#888; font-size:0.9em;">（{{ post.date | date: "%Y-%m-%d" }}）</span>
      </li>
    {% endfor %}
  </ul>
{% endfor %}

<p style="margin-top: 2em;"><a href="/">← 返回首页</a></p>
```

**关键字段说明**：
- `layout: page`：告诉 Jekyll 这是一个独立页面，使用 Minima 的 `page` 布局。
- `title: 标签`：页面标题，会显示在浏览器标签栏和页面顶部。
- `permalink: /tags/`：页面的访问路径，即 `https://你的博客域名/tags/`。

### 第三步：在 `_config.yml` 中注册导航

打开 `_config.yml`，找到 `header_pages:` 配置项（如果没有，就手动添加），把 `tags.md` 加进去：

```yaml
header_pages:
  - index.md      # 首页
  - about.md      # 关于我
  - categories.md # 分类
  - tags.md       # 标签（新增）
```

**注意**：
- 每一行以 `-` 开头，后面跟一个空格，然后是文件名。
- 顺序决定导航栏从左到右的显示顺序。
- 缩进必须是 2 个空格（不要用 Tab）。

### 第四步：提交推送

保存所有文件，执行：

```bash
git add .
git commit -m "新增标签导航页"
git push
```

等待 2 分钟，让 GitHub 构建完成，强制刷新博客，导航栏就会出现“标签”这个 Tab。

## ✏️ 进阶操作：修改导航栏显示的名称

如果你想把导航栏里显示的“分类”改成“文章归类”，或者把“首页”改成“主页”——**只需修改对应页面头部的 `title` 字段，无需改动 `_config.yml`**。

示例：修改 `categories.md` 头部

```yaml
---
layout: page
title: 文章归类      # ← 改这里
permalink: /categories/
---
```

| 导航项 | 对应文件 | 修改 `title` 字段即可 |
| :--- | :--- | :--- |
| 首页 | `index.md` | `title: 主页`（举例） |
| 关于我 | `about.md` | `title: 关于`（举例） |
| 分类 | `categories.md` | `title: 文章归类`（举例） |
| 标签 | `tags.md` | `title: 话题`（举例） |

修改后提交推送，导航文案就会更新。注意 **只改 `title`，不要改 `permalink`**，否则页面会 404。

## 其他类型页面的模板参考

### 1. “友链”页（友情链接）

创建 `friends.md`：

```markdown
---
layout: page
title: 友链
permalink: /friends/
---

## 友情链接

- [程序员鱼皮](https://example.com) - 编程学习
- [张三的博客](https://example.com) - 生活记录
- [李四的技术笔记](https://example.com) - 技术分享

欢迎交换友链，联系我：你的邮箱
```

### 2. “归档”页（按年份汇总所有文章）

创建 `archive.md`：

```markdown
---
layout: page
title: 归档
permalink: /archive/
---

## 文章归档

{% assign posts_by_year = site.posts | group_by_exp: "post", "post.date | date: '%Y'" %}
{% for year in posts_by_year %}
  <h3>{{ year.name }}</h3>
  <ul>
    {% for post in year.items %}
      <li>
        <a href="{{ post.url | relative_url }}">{{ post.title }}</a>
        <span style="color:#888; font-size:0.9em;">（{{ post.date | date: "%Y-%m-%d" }}）</span>
      </li>
    {% endfor %}
  </ul>
{% endfor %}

<p style="margin-top: 2em;"><a href="/">← 返回首页</a></p>
```

### 3. “项目”页（展示作品集）

创建 `projects.md`：

```markdown
---
layout: page
title: 项目
permalink: /projects/
---

## 我的项目

### 项目名称一
- 描述：这是一个什么项目
- 技术栈：HTML, CSS, JavaScript
- 链接：[查看项目](https://example.com)

### 项目名称二
- 描述：另一个项目
- 技术栈：Python, Django
- 链接：[查看项目](https://example.com)
```

## 完整的 `header_pages` 配置示例

假如你的博客有首页、关于我、分类、标签、友链、归档六个页面，`_config.yml` 中的配置就是：

```yaml
header_pages:
  - index.md
  - about.md
  - categories.md
  - tags.md
  - friends.md
  - archive.md
```

导航栏会从左到右依次显示这些页面的 `title`。

## 常见问题与注意事项

### Q1：新增页面后导航没出现？
- 检查 `_config.yml` 中的 `header_pages` 缩进是否正确。
- 确认页面文件的头部（Front Matter）包含 `layout: page` 和 `permalink`。
- 检查文件名是否与 `header_pages` 中填写的完全一致（包括大小写）。

### Q2：修改了 `title` 但导航没变？
- 确保修改的是 `title` 字段，而不是文件名或 `permalink`。
- 提交后等待 2 分钟构建完成，并强制刷新浏览器清除缓存。

### Q3：页面内容不显示，只有标题？
- 检查页面文件中是否包含 `{{ content }}`。如果使用 `layout: page`，内容会自动显示，无需额外配置。

### Q4：我想调整导航菜单的顺序？
- 调整 `header_pages` 列表的顺序即可，从上到下对应从左到右。

### Q5：我想在导航里加一个外部链接（比如我的 GitHub）？
- Minima 的 `header_pages` 只支持内部页面。如果你想加外部链接，需要修改 `_includes/header.html` 文件，手动添加 HTML 代码。但那样会破坏 Minima 的自动导航机制，不建议新手操作。如果确实有需求，可以搜索“Minima 主题自定义导航外部链接”获取教程。

### Q6：新增页面后，其他页面显示正常，但这个新页面 404？
- 检查 `permalink` 路径是否与其他页面冲突。
- 确认文件是否在根目录，而不是在 `_posts` 文件夹里。

## 总结

在 Minima 主题中，管理导航菜单的核心就是 **`header_pages` 配置 + 页面文件的 `title` 字段**：

- **新增导航**：创建页面 → 添加头部信息 → 在 `_config.yml` 中注册 → 提交推送
- **修改名称**：直接改对应页面头部的 `title` 字段 → 提交推送

整个过程不需要接触任何 HTML 或 CSS 代码，全部通过 Markdown 和配置文件完成。以后你想加任何独立页面（“标签”、“友链”、“归档”、“项目”等），或调整导航显示文案，都按这个流程操作即可。
