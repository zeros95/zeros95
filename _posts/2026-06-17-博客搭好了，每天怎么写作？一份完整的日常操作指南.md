## 适用场景

你已经完成了博客的搭建和配置，现在想在本地电脑上写文章并发布到博客上。这篇文章会告诉你：

- 每次写作需要执行哪些步骤
- 有哪些容易忽略的细节
- 如何避免常见的“坑”

---

## 前提条件

在开始之前，请确认你已经完成以下准备：

1. **本地已克隆博客仓库**（如果还没有，执行 `git clone https://github.com/你的用户名/你的用户名.github.io.git`）
2. **安装了文本编辑器**（推荐 VSCode，也可以用记事本）
3. **安装了 Git**（如果之前用 Gridea 时已安装，则已具备）

---

## 📝 日常写作标准流程（三步走）

### 第一步：创建文章文件

在本地仓库的 `_posts` 文件夹中，新建一个 `.md` 文件，文件名必须遵循以下格式：

```
YYYY-MM-DD-文章标题.md
```

**示例**：
- `2026-06-17-我的学习笔记.md`
- `2026-06-18-读书心得-百年孤独.md`

**关键规则**：
- 日期和标题之间**必须有一个短横杠 `-`**
- 标题部分建议用英文横杠连接，避免中文和特殊字符（但也可以用中文）

---

### 第二步：编写文章内容

在新建的文件中，写入以下内容（这是文章模板）：

```markdown
---
layout: post
title: 文章标题
date: 2026-06-17
categories: 分类名称
tags: [标签1, 标签2]
---

正文内容开始...

## 一级标题

这是正文。

### 二级标题

更多内容。
```

**必填字段**：
- `layout: post`：告诉 Jekyll 这是一篇文章
- `title`：文章标题，显示在网页上
- `date`：发布日期，决定文章排序（**建议只写日期，不写时间**）

**可选字段**：
- `categories`：分类，用于“分类”导航页
- `tags`：标签，用于“标签”导航页

**关于日期的重要提醒**：
- 建议写成 `date: 2026-06-17`（不带时分秒），避免 UTC 时区导致的“未来日期”问题
- 具体原因详见本系列第二篇文章

---

### 第三步：提交并推送

在本地仓库文件夹中，打开终端（Git Bash 或 VSCode 终端），依次执行三条命令：

```bash
git add .
git commit -m "新增：文章标题"
git push
```

**提交信息建议**：
- 用中文写清楚本次操作的内容，比如 `"新增：我的学习笔记"`
- 如果是修改错别字，写 `"修正：文章标题中的错别字"`
- 清晰的信息有助于以后回顾历史记录

---

### 第四步：等待并验证

1. 推送后，等待 **2 分钟左右**，让 GitHub 完成构建。
2. 打开浏览器，访问你的博客 `https://你的用户名.github.io`。
3. **强制刷新**（`Ctrl + Shift + R`），确认新文章出现在首页列表中。

## 🔄 多设备写作（台式机 + 笔记本）

如果你在多台电脑上写文章，每次写之前先拉取最新版本：

```bash
git pull        # 先获取另一台设备推送的更新
# 然后写文章、保存文件
git add .
git commit -m "新增：xxx"
git push
```

**为什么这样做**：
- 如果两台设备都修改了同一个文件，`git pull` 可以提前发现冲突
- 如果只是新增不同的文件，`git pull` 能确保本地有最新的文章列表

## 🖼️ 如何插入图片？

1. 在仓库根目录下找到 `images` 文件夹（如果没有，新建一个）。
2. 把图片文件（如 `photo.jpg`）拖进 `images` 文件夹。
3. 在文章中用以下方式引用图片：

```markdown
![图片描述](/images/photo.jpg)
```

**注意**：
- 图片路径以 `/` 开头，表示从仓库根目录开始
- 建议图片体积不要太大（单张不超过 500KB），避免仓库体积膨胀

## ⚠️ 常见错误与预防

| 错误情况 | 预防方法 |
| :--- | :--- |
| 文章不显示 | 检查文件名格式（`YYYY-MM-DD-标题.md`）和文章头部（Front Matter） |
| 文章显示但日期不对 | 检查 `date` 字段，不要写具体时间（如 `16:30`），只写日期 |
| 图片不显示 | 检查图片路径是否以 `/images/` 开头，图片是否已提交推送 |
| 推送失败（403） | 检查 GitHub 令牌是否有效，或网络是否正常 |
| 推送失败（冲突） | 先执行 `git pull`，再重新推送 |

---

## 📋 快速自查清单

在提交推送前，快速检查以下事项：

- [ ] 文件名是 `YYYY-MM-DD-标题.md` 格式
- [ ] 文章头部包含 `layout: post`、`title`、`date`
- [ ] `date` 只写了日期，没有写时间（如 `2026-06-17`）
- [ ] 正文内容保存完成
- [ ] 提交信息写清楚了（如 `"新增：我的学习笔记"`）

---

## 🎯 写作建议

1. **先写起来，再完善格式**：不用一开始就追求完美排版，内容是最重要的。
2. **定期推送**：写完后及时推送，避免本地文件丢失。
3. **保持文件名规范**：养成 `YYYY-MM-DD-标题.md` 的命名习惯，以后文章多了也好管理。
4. **善用分类和标签**：写文章时顺便加上 `categories` 和 `tags`，方便读者按主题浏览。

## 总结

日常写作的核心流程就是三步：

> **创建文件 → 写内容 → `git add`、`commit`、`push`**

这三步重复无数次，你的博客就会越来越丰富。没有复杂的配置，没有额外的工具，你只需要一个文本编辑器和终端就够了。

至此，你的博客从搭建到日常使用，已经有了完整的闭环。接下来就是——**持续写作**。🎉
