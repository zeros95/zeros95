## 为什么想搞一个自己的博客？

之前一直在用飞书文档和公众号写东西，但慢慢发现几个痛点：

- **飞书**：适合写草稿和内部协作，但没法对外公开，离职了数据也可能带不走。
- **公众号**：发出去的文章只能改 20 个字，错别字都没法改，而且所有文章归平台所有。

我想要一个**完全属于自己、永久保存、可以随意修改、不限篇幅**的地方来记录学习、生活和思考。

于是，**GitHub Pages + 本地 Git** 这个方案进入了我的视野。

---

## 最终方案长什么样？

简单来说就是三个东西：

1. **GitHub 仓库**：存放所有博客文章和图片（免费，1GB 空间）
2. **GitHub Pages**：自动把仓库内容渲染成网页（免费）
3. **本地 Git**：在电脑上写文章，用命令一键同步到网上

整个过程**不需要买服务器、不需要写后端代码、不需要装任何第三方博客软件**。

## 踩坑实录（这才是本文精华）

### 坑 1：Gridea 软件连不上 GitHub（浪费 1 小时）

**过程**：
在网上搜“小白搭建博客”，大部分教程都推荐用 **Gridea**（一个带界面的博客客户端）。我按照教程下载安装，填了仓库名、用户名、Token，结果每次点“检测远程连接”都报错：

远程连接失败，请检查仓库、用户名和令牌设置


**尝试了**：
- 改了 8 遍大小写
- 重新生成了 3 次 Token
- 换了 Wi-Fi、换了手机热点
- 把仓库名从 `MorningLigt.github.io` 改到 `morningligt.github.io`（全小写）

**结果**：还是连不上。

**真相**：Gridea 在国内访问 GitHub API（`api.github.com`）极不稳定，跟配置无关，纯粹是网络问题。很多国内用户都卡在这一步。

**教训**：如果某个软件在国内连不上 GitHub，不要死磕。换一条路。

---

### 坑 2：GitHub 仓库名大小写到底怎么写？（纠结 30 分钟）

**过程**：
我的 GitHub 用户名是 `MorningLigt`（有大写字母）。Gridea 里有两栏：

- **仓库用户名**：填 `MorningLigt`（保持原样）
- **仓库名称**：填 `morningligt.github.io`（**必须全小写**）

这是因为 GitHub 在系统底层把仓库路径强制转成了小写。如果填成 `MorningLigt.github.io`，Gridea 就找不到这个仓库。

**教训**：账号名该大写就大写，但仓库名里的用户名部分**必须全小写**。

---

### 坑 3：Gridea 下载太慢（20 分钟）

**过程**：
从 Gridea 官网（`gridea.dev`）下载安装包，速度只有几十 KB/s，等了好久才下完。

**备选方案**：
- 腾讯软件中心搜索 `gridea`（国内镜像，速度快）
- GitHub Releases 页面下载
- 百度网盘（别人存好的安装包）

**教训**：国外软件下载慢是常态，优先找国内镜像。

---

### 坑 4：Token 找不到路径（10 分钟）

**过程**：
按照教程去找 Token 生成页面，结果在 GitHub 设置里迷路了。教程说"在 Settings 里找 Developer settings"，但我翻遍了左侧菜单都没找到。

**坑中坑**：GitHub 里有两个"Settings"！
- 一个是**仓库的 Settings**（在你仓库页面顶部的标签栏里，和 Code、Issues、Pull requests 并列）
- 另一个是**账号的 Settings**（在页面右上角点击你的头像后，下拉菜单里的 Settings）

我当时一直盯着**仓库的 Settings** 翻来翻去，自然找不到 Developer settings。**Developer settings 只存在于账号 Settings 里，仓库 Settings 里根本没有这个选项。**

**正确路径**：
点击页面右上角的**头像** → 在下拉菜单中选择 **Settings**（这是账号设置）→ 左侧菜单滚动到最底部 → **Developer settings** → **Personal access tokens** → **Tokens (classic)** → **Generate new token (classic)**

**教训**：GitHub 的设置菜单层级很深，而且有两个不同的 Settings 入口。以后再找任何账号级别的设置（Token、SSH keys、安全设置等），都从右上角**头像**进去；找仓库相关的设置（Pages、分支保护、Collaborators 等），才从仓库页面的 Settings 标签进去。

## 最终成功方案（放弃 Gridea，改用网页提交）

在 Gridea 连不上之后，我换了一条路：**直接通过 GitHub 网页写文章**。

### 步骤 1：创建博客仓库

在 GitHub 上新建一个仓库，名字必须是：你的用户名.github.io

（注意：用户名部分全小写）

### 步骤 2：用网页提交第一篇文章

在仓库里点击 `Add file` → `Create new file`，文件名写 `index.md`，内容写：
### Hello World
### 这是我的第一篇博客。
