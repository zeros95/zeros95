---

## 什么是分支？用大白话解释

想象你正在写一篇作文，已经写了 800 字，但你想尝试一个完全不同的开头。

**如果没有分支**：你只能直接修改原稿，如果改完觉得不如原来的好，你只能凭记忆恢复，或者从头重写。

**如果有分支**：你可以在不碰原稿的情况下，复制一份"草稿版"出来，在副本上任意修改。如果改得好，就把副本的内容合并回原稿；如果改得不好，直接扔掉副本，原稿毫发无损。

在 Git 中，这个"原稿"就是 **`main` 分支**（你的线上博客），这个"副本"就是**你新建的分支**。

---

## 分支到底有什么用？

对于你的博客来说，分支最常见的用途是：

1. **实验新功能**：比如你想改首页布局、换主题、加新功能，又怕改坏了影响线上博客。建一个新分支，随便折腾，不满意就删掉。
2. **多方案对比**：你可以建两个分支，分别试不同的改法，对比效果后再决定用哪个。
3. **安全备份**：在分支上做任何改动都不会影响你的线上博客，相当于多了一层保险。

## 实际操作：从零开始用分支

### 方式一：使用终端命令行（适合习惯敲命令的人）

#### 1. 查看当前分支

打开终端，进入你的博客仓库目录，执行：

```bash
git branch
```

终端会显示当前所有分支，**带 `*` 号的是你当前所在的分支**。默认情况下，你会看到：

```
* main
```

#### 2. 新建一个分支并切换过去

```bash
git checkout -b 分支名称
```

比如你想实验首页改版，可以建一个叫 `experiment-homepage` 的分支：

```bash
git checkout -b experiment-homepage
```

执行后，你已经自动切换到了这个新分支。

#### 3. 在分支上做改动并提交

现在你可以随意修改文件了（比如改 `index.md`、换头像、调样式），所有改动都只影响这个分支。

改完后，正常提交推送：

```bash
git add .
git commit -m "实验：首页改版"
git push origin experiment-homepage
```

#### 4. 切换回 `main` 分支

```bash
git checkout main
```

切换后，你会发现刚才的改动"消失"了——其实它们还在 `experiment-homepage` 分支里，只是你换了"房间"看不到而已。你的线上博客也完全不受影响。

#### 5. 合并改动到 `main`

如果你对实验结果满意，想让它上线，就切换到 `main` 分支，然后执行合并：

```bash
git checkout main
git merge experiment-homepage
```

合并后，`main` 分支就有了你在实验分支上做的所有改动。然后推送上线：

```bash
git push origin main
```

#### 6. 删除实验分支（可选）

确认合并成功后，可以删除不再需要的实验分支：

```bash
git branch -d experiment-homepage          # 删除本地分支
git push origin --delete experiment-homepage  # 删除远程分支
```

### 方式二：在 GitHub 网页上操作（适合不想用终端的人）

如果你不熟悉命令行，完全可以在 GitHub 网页上完成分支的所有操作。

#### 1. 查看当前分支

打开你的仓库页面 `https://github.com/MorningLigt/morningligt.github.io`。

在左上角文件列表的上方，你会看到一个显示当前分支名称的下拉按钮（默认显示 `main`）。**位置**：就在"Code"按钮的左侧。

**作用**：点击它，你会看到所有分支的列表，以及当前所在的分支（带勾选标记）。

#### 2. 创建新分支

1. 点击左上角的 `main` 下拉按钮。
2. 在出现的搜索框中输入你想要的新分支名称，比如 `experiment-homepage`。
3. 点击下方出现的 **"Create branch: experiment-homepage from main"** 选项。

**注意**：创建后，页面会自动刷新，并切换到新创建的分支。此时你看到的仓库内容依然是完整的，但所有改动都会记录在这个新分支上。

#### 3. 切换分支

1. 点击左上角的 `main`（或当前分支名）下拉按钮。
2. 从下拉列表中找到你要切换到的分支名（比如 `experiment-homepage`），点击它。
3. 页面会自动刷新，文件列表会显示该分支下的最新内容。

**提示**：切换分支后，你依然可以查看和编辑所有文件，但修改的内容会归属到当前选中的分支。

#### 4. 在分支上修改文件

1. 点击文件列表中的任意文件（比如 `index.md`），点击右上角的铅笔图标（✏️）进入编辑模式。
2. 修改内容后，填写提交信息（Commit message），然后点击 **"Commit changes"** 按钮。
3. 此时提交的改动**只会出现在当前分支上**，不会影响 `main` 分支。

#### 5. 通过 Pull Request 合并到 `main`

在 GitHub 网页上，合并分支更推荐使用 **Pull Request（PR）** 功能，它更直观且支持审阅。

1. 点击仓库上方的 **"Pull requests"** 标签页。
2. 点击绿色的 **"New pull request"** 按钮。
3. 在 **"base"** 处选择 `main` 分支，在 **"compare"** 处选择你的实验分支（如 `experiment-homepage`）。
4. 确认显示的分支差异无误后，点击 **"Create pull request"**。
5. 填写标题和描述（可简要说明改动内容），然后点击 **"Create pull request"**。
6. 如果你不需要别人审阅，直接点击页面下方的 **"Merge pull request"**，然后确认 **"Confirm merge"**。
7. 合并成功后，你可以选择 **"Delete branch"** 按钮删除实验分支。

**优点**：这种方式可以清晰地看到改动对比，也方便以后回溯。

#### 6. 在网页上删除分支

1. 点击仓库顶部的 **"Code"** 标签页，回到文件列表。
2. 点击左上角的 `main` 下拉按钮，查看所有分支列表。
3. 找到你想删除的分支，点击分支名称右侧的垃圾桶图标（需是仓库管理员或创建者）。
4. 确认删除即可。

### 📌 网页操作 vs 命令行操作 对比

| 操作 | GitHub 网页端 | 本地命令行 |
| :--- | :--- | :--- |
| **创建分支** | 点击 `main` 下拉，输入新分支名并创建 | `git checkout -b 新分支名` |
| **切换分支** | 点击 `main` 下拉，从列表中选择 | `git checkout 分支名` |
| **提交改动** | 编辑文件后，直接点击"Commit changes" | `git add .` + `git commit -m "信息"` |
| **合并分支** | 通过 Pull Request 按钮合并 | `git merge 分支名` |
| **删除分支** | 在分支列表点击垃圾桶图标 | `git branch -d 分支名` |

## 什么时候该用分支？什么时候不用？

| 场景 | 是否需要用分支 |
| :--- | :--- |
| 写一篇新文章 | ❌ 不需要，直接在 `main` 上写就行 |
| 修改一篇文章的错别字 | ❌ 不需要 |
| 调整博客配色、字体 | ✅ 建议用分支，先看效果再决定 |
| 更换主题或改首页布局 | ✅ 强烈建议用分支，万一不满意可以轻松恢复 |
| 尝试复杂的自定义功能 | ✅ 必须用分支，避免改坏线上环境 |

---

## 为什么我建议你学会用分支？

你现在的博客已经稳定了，但未来你可能会想尝试新东西——比如换主题、加搜索、改配色。如果你直接在 `main` 上改，万一改坏了，你的博客可能暂时无法访问（直到你修复并推送）。而用分支，你可以**先试，满意再上线，不满意直接丢弃**。

**分支 = 试错成本为零的保险箱。**

---

## 一个真实的例子

今天我在做首页改版时，就用了一个分支：

1. 创建 `experiment-homepage` 分支
2. 在分支上改了 `index.md` 和 `style.scss`
3. 本地预览，效果满意
4. 合并到 `main` 分支，推送上线
5. 删除实验分支

整个过程，线上博客一直正常访问，没有受到任何影响。这就是分支的价值。

---

## 总结

**分支是 Git 最友好的功能之一**——它不要求你懂代码，只需要你理解"复制一份副本，在副本上操作，满意再合并"这个逻辑。对于博客维护，它意味着：你可以放心地"折腾"，不用担心"搞砸"。

记住三句话就够了：
- **建分支** = 复制一个平行宇宙
- **切分支** = 换一个房间做事
- **合并分支** = 把房间里的好东西搬回主厅

如果你还没试过分支，下次想改博客样式时，先建一个分支试试吧。🎉
