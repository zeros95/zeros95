## 为什么要学 Markdown？

你的博客使用 Markdown 来写文章。它是一种轻量级标记语言，让你用纯文本就能写出结构清晰、排版美观的文章，而不需要像 Word 那样一边打字一边调整格式。

**核心思想**：你只管写内容，格式由"符号"自动生成。

---

## 常用语法速查表

以下是你写博客时 **80% 会用到** 的语法，按使用频率排序：

---

### 1. 标题

用井号 `#` 表示标题层级：

- `# 一级标题`（文章大标题）
- `## 二级标题`（章节标题）
- `### 三级标题`（小节标题）

**实际效果**（在浏览器中会显示为不同大小的粗体字）：

# 一级标题`（文章大标题）
## 二级标题`（章节标题）
### 三级标题`（小节标题）

---

### 2. 加粗、斜体、删除线

- `**文字**` → 显示为 **文字**
- `*文字*` → 显示为 *文字*
- `~~文字~~` → 显示为
~~文字~~

---

### 3. 列表

**无序列表**（使用 `-` 加空格）：

```
- 第一项
- 第二项
  - 缩进的子项（前面加两个空格）
- 第三项
```
**实际效果**：

- 第一项
- 第二项
  - 缩进的子项
- 第三项

**有序列表**（使用 `1.` `2.` 加空格）：

```
1. 第一步
2. 第二步
3. 第三步
```

---

### 4. 链接

语法：`[显示的文字](网址)`

示例：`[我的博客](https://morningligt.github.io)` → 显示为可点击的 [我的博客](https://morningligt.github.io)

---

### 5. 图片

语法：`![替代文字](图片路径)`

示例：`![我的头像](/images/avatar.jpg)`

> 说明：图片路径可以是网络 URL，也可以是仓库内的相对路径。建议把图片统一放在 `/images/` 文件夹下。

---

### 6. 引用块

在行首使用 `>` 加空格：

```
> 这是一段引用内容。
> 可以换行继续写。
```

**实际效果**：

> 这是一段引用内容。
> 可以换行继续写。

---

### 7. 代码块

用三个反引号（键盘上数字 1 左边的键）包裹代码：

````
```
timezone: Asia/Shanghai
permalink: /:year/:month/:day/:title/
```
````

**行内代码**（用于在段落中标记文件名、命令等）：使用单个反引号包裹，例如  _config.yml  会显示为 `_config.yml`。

---

### 8. 分隔线

单独一行使用三个短横杠 `---`：

```
---
```

**实际效果**：

---

---

### 9. 表格

```
| 列1 | 列2 | 列3 |
| :--- | :--- | :--- |
| 内容A | 内容B | 内容C |
| 内容D | 内容E | 内容F |
```

**实际效果**：

| 列1 | 列2 | 列3 |
| :--- | :--- | :--- |
| 内容A | 内容B | 内容C |
| 内容D | 内容E | 内容F |

---

### 10. 任务列表（待办清单）

```
- [ ] 未完成的任务
- [x] 已完成的任务
```

**实际效果**：

- [ ] 未完成的任务
- [x] 已完成的任务

---

## 你的文章头部（Front Matter）必须写什么？

除了正文，每篇文章开头都要有一块元数据，用 `---` 包裹。**注意：这三行 `---` 是真实的分隔符，不是示例。**

```
---
layout: post
title: 文章标题
date: 2026-06-17
categories: 分类名称
tags: [标签1, 标签2]
---
```

**各字段说明**：

| 字段 | 作用 | 是否必填 |
| :--- | :--- | :--- |
| `layout: post` | 告诉 Jekyll 这是一个文章页面 | ✅ 必填 |
| `title` | 文章标题，显示在网页上 | ✅ 必填 |
| `date` | 发布日期，决定文章排序 | ✅ 必填 |
| `categories` | 分类，用于归档 | ❌ 可选 |
| `tags` | 标签，用于检索 | ❌ 可选 |

> ⚠️ 关于日期：**只写日期，不写具体时间**（如 `date: 2026-06-17`），避免触发 Jekyll 的"未来日期"跳过机制。具体原因详见本系列第二篇文章。

## 文件名规范（务必遵守）

`_posts` 文件夹里的文章文件名必须遵循：

`YYYY-MM-DD-标题.md`

**关键规则**：
- 日期和标题之间**必须有一个短横杠 `-`**
- 标题部分建议用英文和横杠（如 `markdown-guide`），避免中文和特殊字符

**正确示例**：
- `2026-06-17-markdown-guide.md`
- `2026-06-17-博客搭建全记录.md`

**错误示例**：
- `2026-06-17markdown-guide.md`（缺少分隔符）
- `2026-06-17-.md`（标题为空）

## 常用的书写习惯建议

| 习惯 | 原因 |
| :--- | :--- |
| 中英文之间加空格（如 `GitHub Pages`） | 提升阅读体验 |
| 代码块前后加空行 | 确保 Markdown 正确渲染 |
| 列表项用 `-` 加空格开头 | 标准且兼容 |
| 链接文字尽量有描述性 | 对读者更友好 |
| 合理使用标题层级（`#` → `##` → `###`） | 文章结构清晰 |

## 总结：Markdown 的核心哲学

> **专注内容，而非格式。**

你不需要像 Word 那样调整字号、颜色、对齐。你只需要记住几个简单的符号，剩下的交给主题和样式自动处理。

---

### 你最需要记住的 5 个符号（能解决 90% 的排版需求）

| 你想实现的效果 | 对应的符号 | 示例 |
| :--- | :--- | :--- |
| **标题** | `#`（井号） | `## 我的心得` → 显示为二级标题 |
| **加粗** | `**`（两个星号） | `**重要**` → 显示为 **重要** |
| **列表** | `-`（短横杠）或 `1.`（数字加点） | `- 苹果` `1. 第一步` |
| **链接** | `[]()`（方括号+圆括号） | `[我的博客](https://xxx)` |
| **代码块** | 三个反引号（键盘数字1左边的键） | 包裹多行代码 |

---

### 次要但常用的 3 个符号

| 你想实现的效果 | 对应的符号 | 示例 |
| :--- | :--- | :--- |
| *斜体* | `*`（一个星号） | `*提醒*` → 显示为 *提醒* |
| ~~删除线~~ | `~~`（两个波浪号） | `~~错误~~` → 显示为 ~~错误~~ |
| 引用块 | `>`（大于号） | `> 名言警句` |

---

### 上手三步走

1. **先写起来**：不用管格式，把文字打出来。
2. **再套符号**：想清楚哪些地方需要标题、加粗、列表，加上对应的符号。
3. **最后预览**：提交推送后，在浏览器里看效果，不对再改。

---

### 记住这个原则

**如果你在 Word 里会这么做，在 Markdown 里就换个方式表达：**

| 在 Word 里 | 在 Markdown 里 |
| :--- | :--- |
| 点击“标题1”按钮 | 在行首加 `# `（井号+空格） |
| 点击“加粗”按钮 | 用 `**` 包裹文字 |
| 点击“项目符号”按钮 | 行首加 `- `（短横杠+空格） |
| 插入超链接 | 用 `[]()` 包裹 |
| 插入图片 | 用 `![]()` 包裹 |

---

**最后一句**：你现在不需要背下所有语法。写文章时忘记某个符号怎么用，打开这篇文章查一下就好。写 3 篇文章之后，这些符号就会变成你的肌肉记忆。😊
