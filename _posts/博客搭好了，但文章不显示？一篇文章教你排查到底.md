> 这篇文章是我的博客搭建系列第二篇。上一篇讲了如何从零搭好博客，这一篇专门记录我遇到的一个"卡住 2 小时"的问题：**博客首页能打开，但文章就是出不来。**
> 如果你也遇到类似情况，按这篇文章的顺序排查，应该能省下不少时间。

---

## 问题现象

博客搭好之后，首页能正常访问，但文章列表始终是空的。

我明明在 `_posts` 文件夹里写了一篇文章，文件名符合规范，内容也写好了，但首页就是不显示。

排查过程如下，每一步都附带了"为什么这么做"和"我当时的真实感受"。

---

## 排查第一步：检查首页文件优先级（常见坑）

**问题**：我访问博客时，看到的是"博客测试成功"的测试页面，而不是文章列表。

**原因**：仓库根目录同时存在 `index.html` 和 `index.md`，而 GitHub Pages 的优先级是 **`index.html` 高于 `index.md`**。测试页面一直霸占着首页。

**解决**：删掉 `index.html`，让 `index.md` 生效。

**教训**：如果你用 `index.md` 做首页，确认根目录下没有 `index.html` 残留。

---

## 排查第二步：检查 `_config.yml` 配置

**问题**：删掉 `index.html` 后，首页显示了一个空白页面，文章列表还是空的。

**原因**：`_config.yml` 里缺少 `permalink`（文章固定链接格式）配置。Jekyll 不知道该以什么格式生成文章页面。

**解决**：在 `_config.yml` 里添加：

```yaml
permalink: /:year/:month/:day/:title/
```

**为什么要有这一行**：它告诉 Jekyll 每篇文章的 URL 格式。比如 `2026-06-17-xxx.md` 会生成 `/2026/06/17/xxx/` 这样的链接。没有它，Jekyll 可能生成默认的 `/posts/xxx/` 格式，导致首页找不到文章。

---

## 排查第三步：检查文章文件名格式

**问题**：配置完 `permalink` 后，首页依然空白。

**原因**：文章文件名里有中文括号和中文内容。某些情况下 Jekyll 可能因为字符编码问题跳过文件。

**解决**：把文件名改成纯英文+横杠：

```
2026-06-17-github-blog-setup-guide.md
```

**教训**：虽然 Jekyll 理论上支持 UTF-8 文件名，但为了安全，最好使用 `YYYY-MM-DD-英文标题.md` 的格式。中文可以写在文章内部的 `title:` 里。

---

## 排查第四步（关键）：查看 GitHub Actions 构建日志

**问题**：以上步骤都做了，文章还是出不来。

**此时我开始看构建日志**：
1. 进入仓库 → `Actions` 标签页
2. 点击最新的一次 `pages build and deployment`
3. 点击 `build` 步骤，查看详细日志

在日志里我终于找到了关键行：

```
Skipping: _posts/2026-06-17-xxx.md has a future date
```

**终于破案了！**

---

## 排查第五步：理解"未来日期"机制

**原因**：文章头部写了：

```yaml
date: 2026-06-17 16:30:00
```

但 GitHub 服务器用的是 **UTC 时间**（比北京时间慢 8 小时）。我在北京时间凌晨提交时，服务器的 UTC 时间还是 **6 月 16 日**。Jekyll 认为这篇文章是"未来的"，所以直接跳过了。

**解决**：把 `date` 改成纯日期格式（不带时分秒）：

```yaml
date: 2026-06-17
```

或者如果希望保留时分秒，请在 `_config.yml` 中添加 `timezone: Asia/Shanghai`，再使用 `date: 2026-06-17 17:30:00`。

**我目前用的是纯日期格式，省心安全。**

---

## 如何彻底避免"未来日期"问题？

### 方法一：在 `_config.yml` 里添加时区配置（推荐）

```yaml
timezone: Asia/Shanghai
```

这样 Jekyll 就会按北京时间来判断日期，不会再把你的文章判为"未来"。

### 方法二：只写日期，不带时分秒（最安全）

```yaml
date: 2026-06-17
```

不带时分秒时，Jekyll 按当天零点处理，只要你在北京时间当天提交，就不会触发未来日期判断。

**我现在用的是方法二，简单可靠。**

---

## 附：排查清单（傻瓜版）

下次遇到"文章不显示"，按这个顺序查：

| 步骤 | 检查内容 | 怎么查 |
| :--- | :--- | :--- |
| 1 | 根目录有 `index.html` 吗？ | 有就删掉 |
| 2 | `_config.yml` 有 `permalink` 吗？ | 没有就加上 |
| 3 | 文章文件名是 `YYYY-MM-DD-英文标题.md` 吗？ | 中文括号和特殊字符容易出问题 |
| 4 | 文章日期是"今天"吗？ | 如果 UTC 时间还是昨天，就被跳过了 |
| 5 | 打开 Actions 日志，搜索 `Skipping` | 有 `future date` 就说明是日期问题 |

---

## 总结

这次排查的核心收获：

1. **`index.html` 优先级高于 `index.md`**，如果用它测试过，记得删掉。
2. **`permalink` 不是可选项**，是必填项。
3. **时区陷阱真实存在**，`timezone: Asia/Shanghai` 可以一劳永逸，但纯日期格式更简单。
4. **GitHub Actions 日志是最好的朋友**，里面藏着你所有问题的答案。
