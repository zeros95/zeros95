## 这些模板是干什么的？

在 Minima 主题中，导航栏的每个 Tab 都对应一个 `.md` 文件。这个文件决定了：
- 点击导航 Tab 后，页面展示什么内容
- 页面的标题是什么
- 页面的访问链接是什么

下面我按“功能型”（自动生成内容）和“内容型”（手动编写内容）分类，给出常用的模板。

---

## 📌 功能型模板（自动从文章抓取数据）

这类模板使用 Liquid 模板语言，会自动从你的博客文章中读取数据并生成列表。**复制后基本无需修改内容**，只需改 `title` 和 `permalink`。

---

### 模板1：分类归档页（按分类分组）

**功能**：把所有文章按分类分组展示，每个分类下显示该分类的所有文章。

**适用场景**：你的文章有 `categories` 字段，希望读者按分类浏览。

**文件名**：`categories.md`

```markdown
---
layout: page
title: 分类
permalink: /categories/
---

## 所有分类

{% for category in site.categories %}
  <h3 id="{{ category[0] | slugify }}">{{ category[0] }}</h3>
  <ul>
    {% for post in category[1] %}
      <li>
        <a href="{{ post.url | relative_url }}">{{ post.title }}</a>
        <span style="color:#888; font-size:0.9em;">（{{ post.date | date: "%Y-%m-%d" }}）</span>
      </li>
    {% endfor %}
  </ul>
{% endfor %}

<p style="margin-top: 2em;"><a href="/">← 返回首页</a></p>
```

---

### 模板2：标签云页（带标签大小）

**功能**：把所有标签显示为“云”，标签字体大小根据文章数量变化（文章越多，字体越大），点击标签跳转到该标签的文章列表。

**适用场景**：你的文章有 `tags` 字段，希望用更直观的方式展示标签。

**文件名**：`tags.md`

```markdown
---
layout: page
title: 标签云
permalink: /tags/
---

## 标签云

{% assign tags = site.tags | sort %}
{% for tag in tags %}
  <span style="font-size: {%
    if tag[1].size > 10
    %}1.8em
    {% elsif tag[1].size > 5
    %}1.5em
    {% else
    %}1.2em
    {% endif
  %}; margin: 5px; display: inline-block;">
    <a href="#{{ tag[0] | slugify }}" style="text-decoration: none;">{{ tag[0] }}</a>
  </span>
{% endfor %}

<hr>

{% for tag in tags %}
  <h3 id="{{ tag[0] | slugify }}">{{ tag[0] }}</h3>
  <ul>
    {% for post in tag[1] %}
      <li>
        <a href="{{ post.url | relative_url }}">{{ post.title }}</a>
        <span style="color:#888; font-size:0.9em;">（{{ post.date | date: "%Y-%m-%d" }}）</span>
      </li>
    {% endfor %}
  </ul>
{% endfor %}

<p style="margin-top: 2em;"><a href="/">← 返回首页</a></p>
```

### 模板3：时间线归档页（按年份倒序）

**功能**：把所有文章按年份分组，每年内按日期倒序排列，适合快速浏览所有文章的时间线。

**适用场景**：希望按时间维度展示所有文章，不关心分类和标签。

**文件名**：`archive.md`

```markdown
---
layout: page
title: 归档
permalink: /archive/
---

## 文章归档

{% assign posts_by_year = site.posts | group_by_exp: "post", "post.date | date: '%Y'" %}
{% for year in posts_by_year %}
  <h3>{{ year.name }}</h3>
  <ul>
    {% for post in year.items %}
      <li>
        <a href="{{ post.url | relative_url }}">{{ post.title }}</a>
        <span style="color:#888; font-size:0.9em;">（{{ post.date | date: "%Y-%m-%d" }}）</span>
      </li>
    {% endfor %}
  </ul>
{% endfor %}

<p style="margin-top: 2em;"><a href="/">← 返回首页</a></p>
```

### 模板4：纯列表页（不分组）

**功能**：把所有文章按日期倒序排列成一个简单列表，没有任何分组。

**适用场景**：想要最简洁的文章列表，不需要分类或年份分组。

**文件名**：`all-posts.md`

```markdown

layout: page

---

## 全部文章
```

## ⚙️ 通用操作步骤

无论使用哪个模板，都需要完成以下步骤才能让导航 Tab 生效：

### 第一步：创建文件
在仓库根目录下新建一个 `.md` 文件，文件名自定（如 `tags.md`、`archive.md`）。

### 第二步：复制模板内容
把对应的模板代码复制进去，**修改 `title` 和 `permalink` 为你想要的**。

### 第三步：在 `_config.yml` 中注册
打开 `_config.yml`，在 `header_pages:` 列表中添加这个文件名：

```yaml
header_pages:
  - index.md
  - about.md
  - categories.md
  - tags.md          # ← 新增
  - archive.md       # ← 新增
  - friends.md       # ← 新增
```

**注意**：顺序决定导航栏从左到右的显示顺序。

### 第四步：提交推送
```bash
git add .
git commit -m "新增XXX导航页"
git push
```

等待 2 分钟构建完成，强制刷新博客，导航栏就会出现新的 Tab。

## 🤖 如果你不会修改模板，可以找 AI 帮忙

以上模板都是基础版本。如果你想要：

- **不同的分组方式**（比如按月份分组、按字数分组）
- **不同的展示样式**（比如卡片布局、带摘要的列表）
- **更复杂的功能**（比如搜索框、随机文章）

你可以把模板代码复制给 AI（比如我），然后告诉它你想要什么效果，AI 会帮你修改代码。

**示例提问方式**：
> “我现在用的是 `archive.md` 这个模板，我想在每篇文章后面显示文章的字数，应该怎么改？”

> “我想把标签云的样式改成圆角彩色标签，而不是字体大小变化，帮我改一下 `tags.md`。”

## 📋 需要同步修改的地方（重要！）

每次新增导航页面，你需要同步修改以下内容：

| 修改位置 | 内容 | 说明 |
| :--- | :--- | :--- |
| **新文件本身** | `title` 和 `permalink` | 确保标题和路径正确 |
| **`_config.yml`** | `header_pages` 列表 | 新增一行文件名 |
| **Git 提交信息** | 写清楚“新增XXX导航页” | 方便以后查看历史记录 |

**不需要修改的地方**：
- 其他已有的导航页面
- 文章文件（`.md`）
- 主题样式（除非你想自定义）

---

## 💡 使用建议

1. **先选 3-4 个最常用的导航**：不要一次性把所有模板都加上，先加“分类”、“标签”、“归档”这三个最实用的。
2. **命名尽量统一**：`categories.md`、`tags.md`、`archive.md` 这种命名方式清晰易记。
3. **`permalink` 和文件名保持一致**：比如 `tags.md` 的 `permalink` 写成 `/tags/`，方便管理。
4. **有特殊需求找 AI**：不用自己死磕 Liquid 语法，把需求描述清楚，让 AI 帮你生成代码。
